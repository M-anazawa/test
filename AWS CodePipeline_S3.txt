■AWS認証情報のセットアップ
GitHub Secretsの設定: AWSのAWS_ACCESS_KEY_IDとAWS_SECRET_ACCESS_KEYをGitHubのリポジトリのSecretsに安全に保存します。
これにより、ワークフローがAWSリソースにアクセスするための認証情報を保持します。
最小権限の原則の適用: 使用するIAMユーザーには、必要最小限の権限を付与します。
S3へのファイルアップロードおよび削除の権限のみを持つポリシーをIAMユーザーにアタッチします。

    // S3アクセス用のポリシー定義
    const s3Policy = new iam.Policy(this, 'GitHubActionsS3AccessPolicy', {
      policyName: 'GitHubActions-S3-Access',
      statements: [
        new iam.PolicyStatement({
          effect: iam.Effect.ALLOW,
          actions: [
            's3:PutObject',
            's3:DeleteObject'
          ],
          resources: [
            'arn:aws:s3:::バケット名/*', // デプロイ対象のS3バケットARN
            'arn:aws:s3:::バケット名' // バケット自体へのアクセス権限
          ]
        })
      ]
    });

    // IAMユーザーの作成
    const user = new iam.User(this, 'GitHubActionsUser', {
      userName: 'new-GitHub-Actions'
    });

    // ポリシーをユーザーにアタッチ
    user.attachInlinePolicy(s3Policy); // 追加したS3アクセスポリシーをアタッチ
  }

このコードスニペットは、GitHub Actionsの実行環境でNode Version Manager（nvm）をインストールし、設定して使用する一連のステップを示しています。
それぞれのコマンドの意味は以下の通りです：

nvmのインストール
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvmがGitHub Actionsの実行環境にインストールされます。
これにより、異なるバージョンのNode.jsをインストールして切り替えることができるようになります。

環境変数の設定
echo 'export NVM_DIR="$HOME/.nvm"' >> $HOME/.bashrc
このコマンドは、NVM_DIR環境変数を設定し、その値をユーザーのホームディレクトリにある.nvmディレクトリとして~/.bashrcファイルに追記します。
NVM_DIRはnvmがインストールされているディレクトリを指し、nvmのスクリプトがこの変数を使用します。

 nvmの初期化スクリプトを.bashrcに追加
echo '[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"' >> $HOME/.bashrc
このコマンドは、$NVM_DIR/nvm.shスクリプトが存在し読み込み可能であれば、それをソースすることでnvmを初期化するコードを~/.bashrcに追加します。
これにより、シェルセッションが開始されるたびにnvmが利用可能になります。
nvm.shは、Node Version Manager（nvm）をインストールする際に自動的に作成されるスクリプトです。
nvmをインストールすると、このスクリプトは$HOME/.nvmディレクトリ内に配置されます。
したがって、nvm.shを事前に手動で準備する必要はありません。

.bashrcの変更を現在のシェルセッションに適用
source $HOME/.bashrc
sourceコマンドは、指定されたファイル（この場合は~/.bashrc）内のコマンドを現在のシェルセッションで実行します。
これにより、上記で追加した環境変数とnvmの初期化コマンドが有効になり、直ちにnvmコマンドを使用できるようになります。

nvmを使用してNode.jsをインストールし、使用する
nvm install
nvm use
nvm installコマンドを実行すると、nvmは現在のディレクトリまたはその親ディレクトリ内に.nvmrcファイルが存在するかを確認します。
ファイルが存在すれば、その中のバージョン番号に基づいてNode.jsをインストールします。
nvm useコマンドも同様に、.nvmrcに記載されたバージョンのNode.jsを現在のシェルセッションで使用するように切り替えます。

このコードスニペットは、GitHub Actionsのワークフローの一部で、
jakejarvis/s3-sync-action@v0.5.1というアクションを使用して、
ビルドされたアプリケーションをAmazon S3バケットに同期（デプロイ）するためのものです。
各パラメータの意味は以下の通りです：

- name: Deploy to S3 for AaaDev
この行は、GitHub Actionsのワークフロー内でのこのステップの名前を指定しています。
ここでは、「Deploy to S3 for AaaDev」という名前をつけていますが、これは単に識別のためのラベルであり、実際の操作には影響しません。

uses: jakejarvis/s3-sync-action@v0.5.1
この行は、GitHub Actionsワークフローで外部アクションを使用することを指定しています。
jakejarvis/s3-sync-action@v0.5.1は、AWSのS3バケットにファイルを同期するためのアクションです。
@v0.5.1は、このアクションの特定のバージョンを指しており、安定した動作を保証するためにバージョンを指定して使用することが推奨されます。

with:
このセクションでは、s3-sync-actionに渡されるパラメータを定義します。

args: --acl public-read --follow-symlinks --delete
--acl public-read: アップロードされるファイルのアクセス制御リスト（ACL）をpublic-readに設定し、ファイルが公開され、誰でも読み取り可能になることを意味します。
--aclオプションで指定できる値は、Amazon S3のアクセス制御リスト（ACL）ポリシーに基づきます。
これらのACLポリシーは、S3上のオブジェクトに対するアクセス権を細かく制御するために使用されます。
public-readはその中の一つで、オブジェクトを公開して誰でも読み取り可能にします。
public-read以外に指定できる主なACLオプションは以下の通りです：
・private: オブジェクトが完全にプライベートであり、オブジェクト所有者のみがアクセスできます。
・public-read: オブジェクトを公開し、誰でもオブジェクトを読み取ることができますが、書き込みは許可されません。
・public-read-write: オブジェクトを公開し、誰でもオブジェクトの読み取りと書き込みができます（非推奨であり、セキュリティ上のリスクがあるため使用は推奨されません）。
・authenticated-read: 認証されたAWSアカウントのユーザーのみがオブジェクトを読み取ることができます。
・aws-exec-read: AWSのサービスとして実行されるコード（例: EC2インスタンス上のアプリケーション）がオブジェクトを読み取ることができます。
・bucket-owner-read: オブジェクトをアップロードしたユーザーとバケットの所有者のみがオブジェクトを読み取ることができます。
・bucket-owner-full-control: オブジェクトをアップロードしたユーザーとバケットの所有者のみがフルコントロール（読み取り、書き込み、削除等）を持ちます。
--follow-symlinks: シンボリックリンクをたどり、リンク先のファイルを同期します。
--delete: S3バケット内で、ソースディレクトリに存在しないファイルを削除します。これにより、バケットの内容がソースディレクトリの内容と完全に一致するようになります。

bucket: バケット名
このパラメータは、ファイルが同期されるAmazon S3バケットの名前を指定します。
ここでは's3-bucket-for-AaaDev'が使用されていますが、実際のS3バケット名に置き換える必要があります。

source_dir: 'build/dist-adh'
このパラメータは、GitHub Actionsが実行されている環境内の、S3に同期したいファイルが含まれているディレクトリのパスを指定します。
この例では、'build/dist-adh'ディレクトリが指定されており、このディレクトリ内のファイルとフォルダがS3バケットに同期されます。
